# stop-slop-ko: 한국어 AI 문체(slop) 제거기

한국어 글에서 AI 티(slop)를 제거한다.

## 먼저: 두 축은 다르다

이 스킬은 **slop 제거**가 목적이다. "바른 한국어"(문법 정확성)와 헷갈리지 마라.

- **slop** — 예측 가능하고, 공허하고, 목소리가 없는 글. 문법은 완벽해도 slop일 수 있다.
- **정확성** — 이중피동, 번역투 같은 규범 문제. 고치면 깔끔해지지만, 다 고쳐도 slop은 그대로 남을 수 있다.

핵심 규칙은 slop을 다룬다. 정확성은 맨 아래 부록으로 분리했다 — 보조 작업이지 본령이 아니다.

---

## 0단계: 원문 레지스터 파악 (규칙 적용 전)

교정을 시작하기 전에 원문이 어떤 문체인지 확인한다. slop 제거 후에도 그 문체 안에 있어야 한다.

| 레지스터 | 신호 | 교정 후에도 남아야 할 것 |
|------|------|------|
| 격식 보고서·공문 | "본 연구", "응답자의", "나타났다" | 격식 어미, 문어체 어휘 |
| 대화체 칼럼·블로그 | "~했어요", "솔직히", 1인칭 감정 서술 | 구어 리듬, 감정 지시어 |
| 개인 에세이·회고 | 시간 흐름, "버텼다", "결국" | 감정 동사, 문장 간 여백 |
| 메모·요약 | 항목형, 짧은 단문 | 압축 형식 유지 |

**원칙:** 격식 보고서를 구어체로 바꾸거나, 에세이를 메모처럼 압축하면 slop 제거가 아니라 레지스터 파괴다. 각 문체에서 slop만 골라낸다.

---

## 철칙: 지우되 만들지 마라 (규칙보다 먼저)

slop 제거는 **삭제**지 **창작**이 아니다. 아래 둘을 어기면 교정이 아니라 변조다. 모든 규칙은 이 둘 안에서만 적용한다.

1. **정보를 보존하라.** 공허한 표현·군더더기만 지운다. 원문의 사실·주장·논점·뉘앙스는 남긴다. 결과물이 원문 길이의 절반 아래로 떨어지면 십중팔구 내용까지 지운 것이다 — 되돌려 확인하라. 짧다고 더 나은 게 아니다.
2. **없는 사실을 지어내지 마라.** "추상어를 구체로 바꿔라"는 *원문에 그 구체가 있을 때만* 적용된다. 원문에 근거(숫자·예시·고유명사)가 없으면 추상어를 **삭제만** 하고 새 사실을 만들지 않는다. "다양한 요인"의 요인이 원문에 없으면 요인을 발명하지 말고 그 구절을 들어낸다.

## slop 제거 규칙 (요약)

아래 11개가 핵심이다. 상세 표현·패턴 목록은 다음 절에 있다.

**직접 말하기**

1. **서론을 선언하지 마라** — `이 글에서는 ~을 다루겠습니다`, `함께 살펴보도록 하겠습니다` 삭제. 첫 문장이 본론.
2. **단정 회피 어미를 제거하라** — `~라고 할 수 있습니다 → ~입니다`. 주장이면 직접 말하고, 불확실하면 "확실하지 않다"고 밝힌다.
3. **공허한 결론을 잘라라** — `앞으로도 지속적인 관심이 필요합니다` 류는 아무것도 안 더한다. 새 내용 없으면 삭제.
4. **독자를 믿어라** — `중요한 것은`, `핵심은`, `무엇보다도` 삭제하고 바로 내용으로.

**구체성·행위자**

5. **사물을 주어로 쓰지 마라** — 데이터가 말하지 않는다. 사람이 한다. 행위자를 문장 앞에.
6. **공허한 수식어·과장 형용사를 삭제하라** — `다양한`, `효율적인`, `놀라운`, `강력한`은 삭제하거나, *원문에 근거가 있을 때만* 구체어로. 근거가 없으면 삭제만(날조 금지).
7. **막연한 권위에 기대지 마라** — `전문가들은 말한다`, `한 연구에 따르면`(출처 없이)은 출처를 밝히거나 삭제.

**구조·리듬**

8. **접속부사를 반으로 줄여라** — `따라서`, `이를 통해`, `나아가`가 단락마다면 AI 티. 단락당 1개 이하.
9. **이분법·양비론을 직접 말로 바꿔라** — `단순히 X가 아니라 Y → Y`. `물론 ~한 측면도 있다. 하지만 ~` 균형 반사를 끊어라.
10. **같은 말을 두 번 하지 마라** — `즉/다시 말해` 부연, `앞서 언급했듯이` 메타담화, `X란 무엇인가?` 정의 도입 삭제.
11. **한 문체로 수렴하지 마라** (가장 중요) — 모든 문장을 짧은 단정문으로 바꾸면 그 자체가 새 slop이다. anti-slop은 특정 문체가 아니라 **다양성**이다. 출력 전 점검: 문장이 셋 이상인데 길이가 다 비슷하면, 또는 원문보다 절반 넘게 짧아졌으면 다시 본다.

## 상세 패턴: 표현 단위

### 단정 회피 어미
주장을 흐리는 어미. 불확실성이 실제 있으면 명시하고, 없으면 직접 말한다.

- `~라고 할 수 있습니다` → `~입니다`
- `~것으로 볼 수 있습니다` → `~입니다` / `~으로 보입니다`
- `~라고 생각됩니다` → `~라고 생각합니다` / `~입니다`
- `~일 수도 있습니다` (근거 없는 가능성) → 삭제하거나 근거 추가
- `~한 경향이 있습니다` → 구체적 수치·예시로
- `~할 수 있을 것입니다` → `~할 것입니다` / `~합니다`
- `~하는 방향으로 검토해볼 수 있을 것 같습니다` → `~를 검토하겠습니다`

### 공허한 강조어
정보 없이 중요해 보이려는 장식어. 삭제하고 내용으로 간다.

`중요한 것은`, `핵심은`, `특히 주목할 점은`, `무엇보다도`, `결국에는`, `사실은`, `실제로는`, `본질적으로`, `근본적으로`, `궁극적으로`

### 공허한 수식어
구체적 내용이 없으면 삭제, 있으면 그걸 직접 써라.

`다양한`, `효율적인`, `체계적인`, `전반적인`, `포괄적인`, `심층적인`, `혁신적인`, `전략적인`, `지속 가능한` → "다양한 요인이 있다"에서 요인이 **원문에 있으면** 나열하고, 없으면 그 구절을 삭제한다. 요인을 지어내지 않는다.

### 감정 과장 형용사
근거 없이 감탄을 유도하는 형용사. 사실로 교체하거나 삭제.

- 놀라운 / 경이로운 → 무엇이 왜 놀라운지 수치·사실로
- 강력한 / 막강한 → 무엇이 어떻게 강한지
- 압도적인 → 비교 대상과 차이를 쓴다
- 혁신적인 / 획기적인 → 무엇이 이전과 다른지
- 완벽한 / 최고의 → 거의 항상 과장. 삭제
- 엄청난 / 어마어마한 → 구체적 규모로

### 접속부사 남발
같은 접속부사가 글 전체에 반복되면 AI 티. 단락당 1개 이하. 지워서 논리가 안 끊기면 지운다 — 대부분 안 끊긴다.

| 인과·결과 | 첨가·나열 | 전환·요약 |
|------|------|------|
| 따라서 / 그러므로 | 또한 / 게다가 | 이를 통해 (LLM 최빈) |
| 그래서 (격식체 반복 주의) | 더불어 / 아울러 | 이러한 맥락에서 |
| 결과적으로 / 그 결과 | 나아가 (결론 직전 습관) | 이와 같이 / 이처럼 |
| 이로 인해 / 이에 따라 | 뿐만 아니라 | 궁극적으로 / 결국 / 한편 |

### 서론 선언
글 내용을 선언하는 문장. 선언 없이 바로 본론으로.

- "이 글에서는 ~을 다루겠습니다" / "오늘은 ~에 대해 이야기해보겠습니다"
- "먼저 ~에 대해 살펴보겠습니다" / "본 글은 ~을 목적으로 합니다"
- "~에 대한 이해를 돕기 위해" / "독자 여러분께 ~을 소개해드리겠습니다"

**`살펴보다` 계열** (문단 넘길 때 붙이는 메타 서술, 거의 항상 삭제):
"살펴보도록 하겠습니다", "알아보도록 하겠습니다", "짚어보고자 합니다", "하나씩 살펴보면", "자세히 들여다보면", "이제 ~를 살펴볼 차례입니다"

### 공허한 결론
마지막 단락에서 아무것도 안 더하는 문장. 삭제하거나 실제 요점으로.

- "이처럼 ~는 중요합니다" / "앞으로도 지속적인 관심이 필요합니다"
- "우리 모두 함께 ~해야 할 것입니다" / "앞으로의 행보가 기대됩니다"
- "여러분도 한번 생각해보시기 바랍니다" / "이 글이 ~에 도움이 되었으면 합니다"

### AI 클리셰 (보고서·칼럼투)
구체적 내용을 추상적 권위로 바꿔치기하는 상투어. 할 말이 있으면 그 말을, 없으면 문장째 삭제.

| 의의·시사점 포장 | 변화·전환 과장 | 시대·흐름 |
|------|------|------|
| ~라는 점에서 의의가 있다 | 새로운 패러다임 | ~의 시대가 도래했다 |
| 시사하는 바가 크다 | 게임 체인저 | 바야흐로 ~ 시대다 |
| 중요한 시사점을 제공한다 | 새로운 지평/장을 열다 | 급변하는 시대 속에서 |
| 의미 있는 결과로 평가된다 | 패러다임의 전환 | 4차 산업혁명 시대를 맞아 |
| ~에 주목할 필요가 있다 | 전환점을 맞이하다 | 그 어느 때보다 중요해지고 있다 |
| 귀추가 주목된다 | | 빠르게 변화하는 환경에서 |

**원칙:** "의의·시사점·패러다임·주목"이 나오면 멈추고 묻는다 — 이 단어 대신 실제 내용을 쓸 수 있나? 거의 항상 쓸 수 있다.

## 상세 패턴: 구조 단위

### 이분법 대조
요점을 직접 말하지 않고 부정-긍정 쌍을 만드는 구조("not X, but Y"의 직역).

`단순히 X가 아니라 Y다` → Y만. `X의 문제가 아니라 Y의 문제다` → Y를 직접. `X를 넘어 Y로` → 과장. `X가 아니다. Y다.`(두 문장 극적 효과) → Y만. **대신:** Y를 바로 말한다.

### 부정 열거
있는 것 전에 없는 것을 나열하는 극적 빌드업. "단순한 기술 도입이 아닙니다. 비용 절감도 아닙니다. 문화의 문제입니다." → "문화의 문제다"로 시작한다.

### 양비론·균형 반사
주장 전에 반대편을 형식적으로 인정하는 버릇. 외교적으로 보이나 결론을 흐린다.

| 패턴 | 문제 |
|------|------|
| "물론 ~한 측면도 있다. 하지만 ~" | 진짜 반론이면 구체적으로, 형식적이면 삭제하고 결론만 |
| "장점도 있고 단점도 있다" | 무게가 같지 않다. 어느 쪽이 큰지 말한다 |
| "한편으로는 ~ 다른 한편으로는 ~" | 판단을 미룬다. 판단을 쓴다 |
| "일각에서는 ~ 반면 ~" | 출처 없는 양쪽 인용. 누구 주장인지 또는 삭제 |
| "~라는 점은 분명하다. 다만 ~" | "다만"으로 매번 빠져나간다 |

**대신:** 반론에 무게가 있으면 정면으로, 없으면 결론을 직접. 균형은 자동 미덕이 아니다.

### 동어반복 부연
같은 내용을 표현만 바꿔 다시 말하는 구조. `즉/다시 말해/바꿔 말하면 ~`, `이는 곧 ~를 의미한다`(자명한 내용), 같은 단락에서 결론 두 번 진술. **대신:** 처음부터 정확히 한 번. 부연이 필요하면 원문이 불명확하다는 신호다.

### 메타담화 / 자기 참조
글이 자기 구조를 가리키는 표현. `앞서 언급했듯이`, `위에서 살펴본 바와 같이`, `뒤에서 자세히 다루겠지만`, `이 절에서는 ~를 정리한다`, 매 단락 `정리하자면`. **대신:** 삭제. 구조는 내용으로 드러나야 한다.

### 정의로 시작하기
본론 대신 사전적 정의로 글머리를 여는 구조. `X란 무엇인가? X는 ~이다`, `~의 사전적 정의는`, `흔히 ~라고 알려져 있다. 하지만 ~`(정의-반전 클리셰). **대신:** 정의가 논점일 때만 정의한다. 아니면 첫 문장이 주장.

### 막연한 권위 호소
출처 없이 권위를 빌리는 구조(사물 주어와 사촌).

| 패턴 | 문제 |
|------|------|
| "전문가들은 ~라고 말한다" | 어떤 전문가인가. 못 대면 직접 책임진다 |
| "한 연구에 따르면 ~" | 어느 연구인가. 출처를 달거나 삭제 |
| "많은 사람들이 ~라고 생각한다" | 근거 없는 다수결. 삭제 |
| "일반적으로 ~로 여겨진다" | 누가 여기는가. 수동태로 행위자를 숨김 |
| "~라는 것이 정설이다" | 정설이면 출처가 있다 |

**대신:** 출처를 밝히거나, 못 하면 "내 판단으로는"으로 직접 떠안는다.

### 사물 주어 / 거짓 행위자
사람의 행동을 사물·개념에 떠넘기는 구조.

`데이터는 말해줍니다` → 분석한 사람이 결론을 낸다. `이 기술은 보여줍니다` → 누가 무엇을 했는지. `역사는 증명합니다` → 역사학자의 해석. `시장은 반응했습니다` → 구매자들이 샀거나 안 샀다. `연구 결과는 시사합니다` → 연구자들이 주장한다. **대신:** 사람을 주어로. "연구팀은 ~라고 결론 내렸다."

### 3항 나열 반복
세 가지 나열을 습관적으로 쓰는 패턴. "A, B, 그리고 C가 중요합니다"가 반복되거나 모든 소제목이 하위 3항을 가지면 강제로 채운 티. **대신:** 두 항목 또는 하나. 세 개가 꼭 필요하면 이유가 있어야 한다.

### 번호 열거 (`첫째/둘째/셋째`)
짧은 글에도 논점을 번호 매기는 버릇. 산문은 번호 없이 흐른다.

| 패턴 | 문제 |
|------|------|
| "첫째, ... 둘째, ... 셋째, ..." (짧은 단락에) | 번호가 내용보다 구조를 강조. 문장으로 잇는다 |
| "먼저, ... 다음으로, ... 마지막으로, ..." | 같은 패턴 변형. 순서가 필수일 때만 |
| "1) ... 2) ..." / "①②③" (산문 중간) | 보고서투. 진짜 목록이면 글머리표로 분리 |
| 항목마다 "~다는 점이다/~라는 것이다"로 끝남 | 직접 서술로 |

**번호가 정당한 경우:** 순서가 결과를 바꾸는 절차, 나중에 "두 번째 항목"이라 되짚는 글, 항목 5개 이상. **대신:** "전략을 짜고, 사람을 붙이고, 6주 안에 결과를 본다" — 번호 없어도 순서는 읽힌다.

### 단락 리듬
모든 단락이 비슷한 길이 → 길이를 섞는다. 모든 단락이 접속부사로 시작 → 절반 삭제. 단락마다 선언적 한 줄로 끝 → 다양하게. 단락마다 소제목 → 흐름이 있으면 소제목 없이.

### 단일 문체 수렴 (과교정) — 가장 놓치기 쉬움
slop을 지우다 모든 문장을 짧은 단정문·명령형으로 바꾸면 또 다른 AI 문체("자신감 넘치는 단문체")가 된다.

| 과교정 징후 | 교정 |
|------|------|
| 모든 문장이 3-5어절 단정문 | 길이를 섞는다. 긴 문장도 필요 |
| 문장마다 마침표로 끊어 극적 효과 | 자연스러운 연결로 |
| 모든 단락이 명령형/선언으로 끝남 | 끝맺음을 다양화 |
| 숫자·통계를 억지로 모든 문장에 | 근거가 있을 때만 |
| 목적(편지·보고서·칼럼)과 무관하게 같은 톤 | 목적에 맞는 목소리를 남긴다 |

**원칙:** 고친 글을 소리 내 읽었을 때 한 가지 리듬으로 똑같이 들리면, 그것도 slop이다. 그리고 원문보다 짧아졌다고 더 나은 게 아니다 — 정보가 사라졌으면 과교정이다. (아래 예시 8·9·10)

### 독자 참여 요청
독자를 끌어들이는 척하는 공허한 패턴. `여러분은 어떻게 생각하시나요?`, `한번 생각해보시기 바랍니다`, `혹시 이런 경험이 있으신가요?` → 삭제, 바로 요점으로.

### 과잉 경어 구조
완곡함을 5중으로 쌓는 책임 회피. `~하는 방향으로 검토해볼 수 있을 것 같습니다` → `~를 검토하겠습니다`. `~하시면 좋을 것 같습니다` → `~하세요`. 동사+수동+가능+추정+격식체를 겹치면 AI 티.

### 수사적 질문
영어 rhetorical setup의 직역. `왜 이것이 중요할까요?`(바로 답하면서) → `이것이 중요한 이유는` 또는 이유부터. `그렇다면 해결책은?` → 해결책을 쓴다.

## Before/After 예시

**예시 1 — 서론 지연 + 단정 회피**
Before: "오늘은 재택근무가 팀 생산성에 미치는 영향에 대해 이야기해보겠습니다. 본 글에서는 두 가지 측면을 살펴볼 것인데, 이를 통해 독자 여러분께서 깊은 이해를 가질 수 있게 되기를 바랍니다. 먼저 커뮤니케이션 측면을 살펴보면, 슬랙 메시지는 늘었지만 실시간 의사결정 속도는 느려졌습니다."
After: "재택근무 이후 슬랙 메시지는 늘었지만 실시간 의사결정 속도는 느려졌다."
(메타 선언 두 문장 전체 삭제. 실제 내용이 담긴 마지막 문장만 남겨 첫 문장으로 세운다.)

**예시 2 — 이중피동 + 번역투**
Before: "이 결과는 여러 연구자들에 의해 검토되어지고 있으며, 향후 더 많은 연구가 필요한 것으로 보여집니다."
After: "여러 연구자가 이 결과를 검토하고 있다. 후속 연구가 필요하다."

**예시 3 — 사물 주어 + 공허한 강조**
Before: "데이터는 우리에게 중요한 사실을 말해줍니다. 무엇보다도, 이 기술은 산업 전반에 혁신적인 변화를 가져올 것으로 생각됩니다."
After: "이 기술은 산업 구조를 바꾸고 있다."
(사물 주어 '데이터는 말해줍니다' 삭제. '무엇보다도'·'혁신적인'·'전반에' 삭제. '생각됩니다' → '있다'로 직접화. 원문에 구체적 수치나 사례가 없으므로 더 이상 구체화하지 않는다 — 숫자를 지어내면 slop 제거가 아니라 날조다. 예시 10 참고.)

**예시 4 — 3항 나열 + 접속부사 남발**
Before: "따라서 우리는 첫째, 전략을 수립하고, 둘째, 팀을 구성하며, 셋째, 실행 계획을 마련해야 합니다. 이를 통해 성공적인 결과를 도출할 수 있을 것입니다. 나아가 이러한 접근은 조직 역량을 강화하는 데 기여할 것으로 기대됩니다."
After: "전략부터 짜고, 사람을 붙여라. 6주면 첫 결과가 나온다."

**예시 5 — 이분법 대조 + 공허한 결론**
Before: "이것은 단순한 기술의 문제가 아닙니다. 비용의 문제도 아닙니다. 이것은 바로 사람의 문제입니다. 이처럼 인간 중심적 접근이 얼마나 중요한지 우리 모두 인식하고, 앞으로도 지속적인 관심을 가져야 할 것입니다."
After: "사람이 문제다. 팀이 바뀌지 않으면 기술이나 예산이 바뀌어도 같은 결과가 나온다."

**예시 6 — 수사적 질문 + 단정 회피**
Before: "그렇다면 왜 이 문제가 반복될까요? 이는 구조적 요인이 있기 때문이라고 할 수 있습니다. 조직 문화, 인센티브 구조, 그리고 의사소통 방식이 복합적으로 작용하는 것으로 볼 수 있습니다."
After: "인센티브 구조가 잘못됐다. 실패를 보고해도 불이익이 없어야 같은 실수가 반복되지 않는다."

**예시 7 — 과잉 완곡 + 명사화**
Before: "이 부분에 대해서는 추가적인 검토가 필요한 것으로 생각되며, 관련 팀과의 협의를 통해 구체적인 방향을 설정하는 것을 고려해볼 수 있을 것 같습니다."
After: "관련 팀과 협의하겠습니다."

**예시 8 — 과교정 주의 (새 slop)**
Before(원래 slop): "이번 프로젝트는 다양한 이해관계자들과의 긴밀한 협력을 통해 성공적으로 마무리될 수 있었으며, 이는 향후 유사 프로젝트에 중요한 시사점을 제공할 것으로 기대됩니다."
과교정(새 slop): "프로젝트 끝났다. 협력이 핵심이었다. 다음에도 쓸 수 있다." ← 전부 3-4어절 단정문, 리듬 죽음.
제대로: "프로젝트를 마쳤다. 마케팅팀이 일정을 3주 당겨준 게 컸다. 다음에 비슷한 일을 한다면, 초기에 팀을 다 모으는 것부터 하겠다." ← 길이 섞임 + 구체적 사실 + 사람의 판단.

**예시 9 — 한 Before, 세 목소리** (de-slop에 정답 문체는 없다)
Before: "고객 만족도 향상을 위해서는 다각도의 접근이 필요하며, 지속적인 모니터링을 통해 개선 방안을 마련해 나가는 것이 중요하다고 할 수 있습니다."
- 보고서 톤: "고객 만족도를 높이려면 응대 속도와 환불 정책 두 가지를 먼저 손봐야 한다. 매월 NPS로 효과를 확인한다."
- 대화체 톤: "만족도를 올리고 싶다면 먼저 응대 속도부터 봐야 합니다. 환불도요. 매달 점수를 재서 뭐가 먹히는지 확인하면 됩니다."
- 메모 톤: "우선순위: (1) 응대 속도, (2) 환불 정책. 효과는 월간 NPS로 추적."

세 버전 모두 slop이 아니다. 추상어를 구체로 바꾸고 단정 회피를 없앤 공통점만 있고, 문체는 목적에 따라 다르다. (단, 위 "응대 속도·환불·NPS"는 원문 맥락에 있던 정보일 때만 쓴다. 원문에 없으면 만들지 않는다.)

**예시 10 — 날조 금지 (원문에 근거가 없을 때)**
Before(slop): "이 기술은 다양한 분야에서 놀라운 성과를 내고 있으며, 향후 더욱 주목받을 것으로 기대됩니다."
나쁜 교정(날조): "이 기술은 의료·물류·금융에서 오진율을 30% 낮췄다." ← 원문에 없는 분야·수치를 지어냈다. slop을 거짓으로 바꿨다.
올바른 교정: "이 기술은 이미 현장에서 성과를 내고 있다." ← 근거 없는 구체는 만들지 않고, 공허한 수식어("다양한·놀라운·주목")만 걷어낸다. 구체가 필요하면 원문 작성자에게 사실을 받아 채운다.

## 빠른 체크리스트

| 확인 | 있으면 |
|------|--------|
| 첫 문장이 구조 선언 | 본론으로 교체 |
| `~라고 할 수 있습니다` `~것으로 볼 수 있습니다` | 직접 주장으로 |
| 마지막 단락이 `앞으로 ~필요합니다` | 삭제 |
| `중요한 것은` `핵심은` `무엇보다도` | 삭제, 바로 내용으로 |
| 사물·개념이 주어인 문장 | 행위자(사람)를 주어로 |
| `다양한` `효율적인` `놀라운` `강력한` | 삭제 또는 구체어 |
| `전문가들은` `한 연구에 따르면` (출처 없이) | 출처 명시 또는 삭제 |
| `시사점을 제공` `~라는 점에서 의의` `새로운 패러다임` | 실제 내용으로 |
| `살펴보도록 하겠습니다` `알아보겠습니다` | 삭제, 바로 내용으로 |
| `따라서` `이를 통해` `나아가` 단락마다 | 절반 삭제 |
| `단순히 X가 아니라 Y` | Y만 남긴다 |
| `물론 ~ 하지만 ~` 형식적 양비론 | 결론만 말한다 |
| `즉` `다시 말해` 동어반복 | 한 번만 |
| `앞서 언급했듯이` `X란 무엇인가?` | 삭제, 바로 본론 |
| `첫째... 둘째... 셋째...` / 세 항목 나열 반복 | 둘 또는 하나로 |
| `여러분은 어떻게 생각하시나요?` | 삭제 |
| 고친 글이 전부 같은 짧은 단정문 | 길이·목소리 다양화 |
| 원문에 없는 사실·숫자·예시가 새로 들어감 | 삭제 (날조 금지) |
| 결과물이 원문 길이의 절반 이하로 짧아짐 | 내용 손실 점검, 사라진 논점 복원 |

## 채점

각 항목 1-10점:

| 항목 | 질문 |
|------|------|
| 직접성 | 주장인가, 선언인가? |
| 리듬 | 문장 길이가 다양한가? |
| 신뢰 | 독자를 존중하는가? |
| 자연스러움 | 사람이 쓴 것처럼 들리는가? |
| 밀도 | 잘라낼 수 있는 부분이 없는가? |
| 목소리 | 한 문체로 납작해지지 않고 목적에 맞는 톤이 남았는가? |
| 보존 | 원문의 사실·논점이 유지됐는가? 없는 사실을 더하지 않았는가? |

49/70 미만이면 다시 쓴다. **보존 항목이 6점 미만이면 점수와 무관하게 다시 쓴다** (정보 손실·날조는 교정 실패다).

## 부록: 바른 한국어 (정확성 축 — slop과 별개)

규범 문제다. 고치면 깔끔해지지만 slop 제거와는 다른 작업이다. slop을 다 잡은 뒤 정리 차원에서 적용한다.

**A. 이중피동을 끊어라**
피동 접사(`-이-/-히-/-리-/-기-`)와 `-어지다`를 겹쳐 쓰지 않는다.

| 쓰지 말 것 | 교체 | 쓰지 말 것 | 교체 |
|------|------|------|------|
| 보여지다 | 보이다 | 되어지다 | 되다 |
| 읽혀지다 | 읽히다 | 여겨지다 | 여기다/보다 |
| 쓰여지다 | 쓰이다 | 알려져 있다(반복) | 알려진 |

**B. 번역투 조사를 줄여라**

- `~에 의해` → 능동문으로 (사물주어 slop 제거와 같은 방향). "전문가에 의해 검토된" → "전문가가 검토한"
- `~의` 연속 → 끊는다. "회사의 전략의 핵심의 문제" → "회사 전략의 핵심 문제"
- `~하는 것` 명사화 → 동사로. "해결하는 것이 중요하다" → "해결해야 한다"
- `~에 있어서` → `~에서`/`~에 관해`
- `~를 통해` (도구·수단) → `~해서`/`~로`. "협력을 통해" → "협력해서"
- `~하고 있습니다`(상태·사실) → `~합니다`/`~중입니다`. "진행되고 있습니다" → "진행 중입니다"

> 한자어 순화(활용→사용, 구현→만든다 등)는 이전 버전에 있었으나 삭제했다. 순화는 slop 신호가 아니고, 강제하면 레지스터를 납작하게 만들어 오히려 새 slop을 유발한다.
